파일과 디렉토리 조작

GUI 환경의 파일관리자로도 손쉽게 조작이 가능한데 커맨드라인 프로그램을 사용하는 이유는 강력함과 유연성 때문이다. 예를들어 어떤 디렉토리에 저장된 모든 HTML 파일을 다른 디렉토리로 복사하는 과정을 단 한 줄로 끝낼 수 있다.

cp -u *.html destination

와일드카드 = 글로빙

  • * : 모든 문자

    • g* : g로 시작하는 모든 파일
    • b*.txt : b로 시작하되 .txt 형식인 파일

  • ? : 모든 문자이되 한 글자를 뜻함

    • Data??? : Data로 시작하면서 뒤에 세 개의 문자만 있는 파일

  • [characters] : characters 문자셋에 포함된 문자

    • [abc]* : a, b, c 로 시작하는 모든 파일

    • BACKUP.[0-9][0-9][0-9] : BACKUP으로 시작하면서 뒤에 세 개의 숫자로 된 파일

  • [!characters] : characters 문자셋에 포함되지 않은 문자

    • [![:digit:]]* : 숫자로 시작하지 않는 모든 파일

  • [[:class:]] : 지정된 문자 클래스에 포함된 문자

    • 가장 많이 사용되는 문자 클래스

      문자클래스 매칭 문자
      [:alnum:] 모든 알파벳과 숫자, 문자
      [:alpha:] 모든 알파벳 문자
      [:digit:] 모든 숫자
      [:lower:] 모든 소문자
      [:upper:] 모든 대문자

    • *[[:lower:]123] : 파일명이 소문자로 끝나거나 1,2,3 으로 끝나는 파일

mkdir

디렉토리를 생성할 때 사용한다.

  • mkdir dir1
  • mkdir dir1 dir2 dir3

cp

파일과 디렉토리를 복사하는 명령어로 두 가지 방법이 있다.

  1. cp item1 item2

    item1이라는 파일 또는 디렉토리를 item2 파일 또는 디렉토리로 복사하는 방법

  2. cp item1 item2 item3 directory

    다수의 파일이나 디렉토리를 다른 디렉토리로 복사하는 방법

  • cp 옵션

    옵션 설명
    -a, –archive 파일 및 디렉토리뿐만 아니라 소유자 및 권한 정보와 같은 속성까지 모두 복사한다 (일반적으로는 복사를 하는 사용자의 기본적인 속성을 복사)
    -i, –interactive 기존 파일을 덮어쓰기 전에 확인 메세지를 보여주는 옵션 이 옵션 없이 cp 명령어를 사용하면 확인 과정 없이 그대로 파일을 덮어씀
    -r, –recursive 디렉토리와 그 안의 내용가지 복사할 때 쓰는 옵션
    -u, –update 어떤 디렉토리에 있는 파일을 다른 디렉토리로 복사할 때, 그 디렉토리에는 없거나 최신 버전인 파일만을 복사하기 위해서 이 옵션을 사용
    -v, –verbose 복사가 완료되었다는 메시지를 보여주는 옵션

  • cp 사용 예제

    • cp file1 file2 : file1을 file2로 복사, file2 파일이 이미 있다면 file1 내용을 그대로 덮어쓰고 없다면 새로 생성됨

    • cp -i file1 file2 : 위와 같이 복사가 이뤄지지만 만약 file2파일이 있다면 덮어쓰기 여부에 대한 확인 메시지를 볼 수 있음

    • cp file1 file2 dir1 : file1, file2를 dir1 디렉토리로 복사 단, dir1 디렉토리는 미리 존재해야 함

    • cp dir1/* dir2 : dir1 디렉토리에 있는 모든 파일 dir2로 복사 단, dir2 디렉토리는 미리 존재해야 함

    • cp -r dir1 dir2 : dir1 디렉토리와 그 안에 있는 모든 내용을 dir2 디렉토리로 복사 dir2가 없으면 새로 생성됨

mv

쓰임새에 따라 파일을 이동할 수도 있고 파일명을 수정할 수도 있는 명령어 두 경우 모두 기존 파일명은 더 이상 존재하지 않게 되며 쓰임은 cp 명령어와 매우 비슷하다.

  • mv 옵션

    옵션 설명
    -i, –interactive 기존 파일을 덮어쓰기 전에 확인 메세지를 보여주는 옵션, 이 옵션없이 사용하면 확인 과정 없이 그대로 파일을 덮어쓰게 됨
    -u, –update 파일을 다른 디렉토리로 이동하려고 할 때, 그 디렉토리에는 없거나 또는 최신 버전인 파일만을 이동하기 위해서 이 옵션을 사용
    -v, –verbose 이동이 완료되었다는 메시지를 보여주는 옵션

  • mv 사용 예제

    • mv file1 file2 : file1 파일을 file2 파일로 이동한다. file2 라는 파일이 이미 있다면 file1 내용을 그대로 덮어쓰게 된다. file2 가 없으면 새로 생성된다 두 경우 모두 file1은 사라진다

    • mv file1 file2 dir1 : file1과 file2를 dir1 디렉토리로 이동한다. 단, dir1 디렉토리는 미리 생성되어 있어야 한다.

    • mv dir1 dir2 : dir1과 그 내용까지 모두 dir2로 이동한다. dir2 디렉토리가 기존에 없다면 새로 생성되고 dir1 내용이 모두 그대로 이동하고 dir1은 삭제된다.

rm

삭제하기 위한 파일 및 디렉토리를 하나 이상 입력하여 사용하는 명령어

※ 리눅스처럼 유닉스형 운영체제는 삭제를 취소할 수 있는 명령어가 없다. rm과 함께 와일드카드를 사용할 때마다 입력을 주의해야 한다 ls 명령어로 먼저 테스트 한 후 삭제하는 것이 좋다.

  • rm 옵션

    옵션 설명
    -i, –interactive 기존 파일을 삭제하기 전에 확인 메세지를 보여주는 옵션, 이 옵션없이 사용하면 확인 과정 없이 그대로 파일을 삭제하게 됨
    -r, –recursive 재귀적으로 디렉토리를 삭제한다. 즉, 삭제된 디렉토리에 하위 디렉토리가 있다 해도 모두 삭제된다
    -f, –force 존자해지 않는 파일은 확인 메시지 없이 무시하라는 옵션 이 옵션은 –interactive 옵션을 완전히 무시해버린다.
    -v, –verbose 삭제가 완료되었다는 메시지를 보여주는 옵션

  • rm 사용 예제

    • rm file1 : file1 파일을 완전히 삭제한다.

    • rm -i file1 : file1 파일을 삭제하기 전에 사용자 확인 메시지를 보여준다.

    • rm -r file1 dir1 : file1 파일과 dir1 디렉토리 및 하위 내용까지 모두 삭제한다.

    • rm -rf file1 dir1 : 위와 같이 file1과 dir1 모두 다 삭제하되 file1 이나 dir1이 존재하지 않더라도 rm 명령어가 실행된다.

ln

하드 링크와 심볼릭 링크를 만들 때 사용한다. 심볼릭 링크란 예를 들어 foo 파일이 현재 버전 2.6이고 foo-2.6 파일명으로 저장되어 있을 때 심볼릭 링크를 하나 생성해서 이것이 foo-2.6 파일을 가리키도록 하면 프로그램이 foo 파일을 사용하려고 할 때 실제 foo-2.6 파일을 열게 하는 기능이다.

  • 버전이 없그레이드 되어 foo-2.7 파일로 시스템에 저장되었다 해도 이전 심볼릭 링크를 삭제하고 foo-2.7 파일을 가리키도록 새롭게 지정해 주면 됨

  • 이로써 모든 버전을 시스템에 저장하여 관리 또한 가능해짐 예를 들어 업그레이드 된 foo-2.7 파일에 버그가 발견되면 다시 이전 버전의 파일로 심볼릭 링크를 수정해주기만 하면 된다.

lrwxrwxrwx 1 root root   11 2020-06-07 20:31 libc.so.6 -> libc-2.6.so

libc.so.6 이라는 심볼릭 링크는 libc-2.6.so 라는 공유 라이브러리 파일을 가리키고 있다.

  1. 하드링크 생성 : ln file Link

  2. 심볼릭 링크 셍성 : ln -s item Link (ln -s [원본 파일 또는 디렉토리] [심볼릭 링크 이름]

하드링크 vs 심볼릭링크

하나의 파일에는 하나의 하드 링크가 있는데 그것이 바로 파일에 이름을 만들어 준다. 하드링크가 만들어질 때 그 파일에 대한 디렉토리가 곧바로 생성된다. 그러나 하드링크는 단점이 두가지가 있다.

  • 파일시스템 외부에 있는 파일을 참조할 수 없다. 다시 말해 하드 링크는 같은 디스크 파티션에 있는 파일이 아니면 참조할 수 없다는 것이다.

  • 하드 링크는 디렉토리를 참조할 수 없다.

하드링크가 삭제될 때 링크도 함께 사라지지만 파일 내용은 그 파일의 모든 링크가 삭제될 때까지 계속 남아있는다. 즉 파일에 할당된 공간이 그대로 남게 된다 이러한 하드 링크의 한계를 극복하기 위해서 심볼릭 링크가 탄생되었다.

심볼릭 링크는 참조될 파일이나 디렉토리를 가리키는 텍스트 포인터가 포함된 특수한 파일을 생성한다. 심볼릭 링크에 편집을 하게 되면 심볼릭 링크가 참조하고 있는 파일도 역시 똑같은 변경이 이루어진다. 하지만 심볼릭 링크를 삭제하는 경우엔 그 링크만 삭제되고 파일은 남아있다. 심볼릭 링크를 삭제하기 전에 파일을 지웠다면 심볼릭 링크는 살아있지만 이 링크는 아무것도 가리키지 않게 된다. 이러한 경우를 링크가 깨졌다고 표현한다. ls 명령어로 인해 빨간색이나 다른 색상으로 깨진 링크를 볼 수 있다.
