Linux Command Line - File system-2

Jun 5, 2020 · 4 min read

시스템 살펴보기

ls

현재 작업 디렉토리에 있는 파일과 하위 디렉토리들이 모두 표시됨

  • ls /usr 현재 작업디렉토리에서 다른 디렉토리의 목록 보기

  • ls ~ /usr 한 번에 여러 디렉토리 목록을 보는 방법으로 홈 디렉토리(~)와 /usr 디렉토리를 확인 하는 명령어

  • ls -l 파일 디렉토리명뿐만 아니라 좀 더 자세한 속성까지 확인 하는 명령어

옵션 long 옵션 설명
-a –all 모든 파일 보기, .(점) 으로 시작하는 숨김파일까지도 표시
-d –directory 보통 ls 명령어에 디렉토리를 명시하면 해당 디렉토리 자체가 아닌 디렉토리 내용을 확인할 수 있는데, 이 옵션을 -l과 함께 사용하면 디렉토리 내용이 아닌 디렉토리 자체 정보를 자세하기 확인 가능하다
-F –classify 지시 문자를 추가로 표시, 예. 디렉토리명이면 끝에 / (슬래시) 덧붙임
-h –human-readable -l 옵션과 함께 사용하여 파일 크기를 사람이 인식하기 쉬운 형태로 표시해줌
-r –reverse 출력 결과를 역순으로 표시, 일반적으로 ls는 알파벳 오름차순 표시
-s 파일 크기순으로 정렬
-t 파일 수정 시간순으로 정렬

명령어 -옵션 명령인자 형태로 사용하며 여러 옵션을 한 명령어에 연이어 사용 가능하다.

file

file filename : 파일 타입을 확인 하는 명령어

[me@linux ~]$ file picture.jpg
picture.jpg: JPEG image data, JFIF standard 1.01

less

텍스트 파일을 볼 때 사용하는 프로그램으로 텍스트 파일을 스크롤하여 확인 할 수 있다.

Ex. 시스템의 모든 사용자 게정 정보가 정의된 파일 확인 :

[me@linux ~]$ less /etc/passwd

Q 키를 누르면 종료 할 수 있다.

  • less 명령키

    명령키 실행
    PAGE UP 또는 b 한 페이지 위로
    PAGE DOWN 또는 스페이스 바 한 페이지 아래로
    ↑ 한 줄 위로
    ↓ 한 줄 아래로
    G 텍스트 파일 맨 마지막으로 이동
    1G 또는 g 텍스트 파일 맨 처음으로 이동
    /문자열 아래 방향으로 진행하며 입력된 문자열 찾기
    n 이전 검색어의 다음 찾기
    h 도움말 보기
    q 프로그램 종료

  • 리눅스 시스템 디렉토리

    디렉토리 설명
    / 루트 디렉토리. 파일 시스템의 시작점
    /bin 시스템 부팅과 실행에 필요한 바이너리(프로그램)들을 포함하고 있다.
    /boot 리눅스 커널. 시작 RAM 디스크 이미지 (시스템 부팅 시 필요한 드라이버가 있음), 부트로더를 포함하고 있다. º /boot/grub/grub.conf 또는 menu.lst : 부트로더 설정 파일 º /boot/vmlinuz, 리눅스 커널
    /dev 디바이스 노드를 갖고 있는 특수한 디렉토리 “리눅스에서는 모두 다 파일이다” 라는 말에는 디바이스도 포함, 이 디렉토리는 커널이 인식하고 있는 모든 디바이스들을 관리
    /etc /etc 디렉토리에는 시스템 전반의 환경설정 파일이 들어있다. 시스템 부팅 시에 ,시스템의 각 서비스를 시작하는 쉘 스크립트 전부가 들어 있다. (모두 텍스트 형식) º /etc/crontab 자동 실행되는 업무를 정의하는 파일 º /etc/fstab 저장장치 테이블과 해당 마운트 포인트 정보 º /etc/passwd 사용자 계정정보
    /home 각 사용자마다 /home 디렉토리를 가지며 일반사용자는 홈 디렉토리내에서만 파일을 편집 가능
    /lib 주요 시스템 프로그램에서 사용하는 공유 라이브러리 파일이 저장되어 있음 (윈도우즈-DLL과 비슷)
    /lost+found 파일시스템에 문제가 생겼을 때 부분적인 복구를 도와준다. 시스템에 심각한 상황이 발생하지 않는다면 이 디렉토리는 늘 비어있다.
    /media USB 드라이버, CD-ROM 등과 같은 휴대용 장치가 시스템에 연결될 때 자동으로 마운트되는 마운트 포인트는 여기에 저장
    /mnt 구식 리눅스 시스템에서 수동으로 휴대용 장치를 제거하기 위한 마운트 포인트를 저장
    /opt 추가적인 소프트웨어를 설치할 때 사용, 주로 상업용 소프트웨어가 설치됨
    /proc 하드 드라이브에 실제 저장된 파일시스템이 아니라 리눅스 커널이 관리하는 가상 파일시스템, 이곳에 저장된 것들은 커널 자체를 들여다 볼 수 있는 파일이다.
    /root 루트 계정의 홈 디렉토리
    /sbin 시스템 바이너리 파일들이 위치, 일반적으로 슈퍼유저를 위한 중요한 시스템 작업을 수행하는 프로그램들이다.
    /tmp 다양한 프로그램들에 의해서 만들어지는 임시 파일을 저장하는 임시 저장용 디렉토리
    /usr 리눅스 시스템에서 가장 큰 트리구조를 갖고있으며 일반 사용자가 사용하는 모든 프로그램과 지원 파일들이 모두 들어있다.
    /usr/bin 사용중인 리눅스 배포판이 설치한 실행 프로그램들이 위치, 여기는 일반적인 수많은 프로그램들을 저장하는 공간으로 사용되지 않는다.
    /usr/lib /usr/bin 디렉토리에 잇는 프로그램을 위한 공유 라이브러리가 있다.
    /usr/local 일반적으로 사용하는 배포판 프로그램들 대신 시스템 전반에 걸쳐 사용되는 프로그램들이 저장되는 공간, 소스 코드로 컴파일된 프로그램은 주로 /usr/local/bin 디렉토리에 설치된다. 최신 리눅스 시스템에서는 이러한 트리가 존재하나 시스템 관리자가 이 디렉토리에 별도의 작업을 하지 않는 이상 비어있는 상태
    /usr/sbin 시스템 관리 프로그램이 위치
    /usr/share /usr/bin 디렉토리에 있는 프로그램이 사용하는 공유 데이터를 저장하며 또한 디폴트 설정 파일, 아이콘, 스크린 배경화면, 음악 파일등이 있다.
    /usr/share/doc 시스템에 설치된 대부분의 패키지에는 문서파일이 포함되어 있는데 이 디렉토리에 그러한 패키지 문서 파일이 저장된다.
    /var /tmp 및 /home 디렉토리를 제외하면 대부분의 디렉토리들은 상대적으로 정적인 디렉토리임, /var 디렉토리 트리는 상대적으로 변하기 쉬운 데이터를 모아두는 공간이다. 다양한 데이터베이스, 스풀 파일, 사용자 메일 등과 같은 데이터가 저장된다
    /var/log 시스템 활동을 기록하는 로그 파일이 들어있다. 매 시간마다 시스템 상황을 확인하는 매우 중요한 기록파일이다. 그 중 유용한 것 하나가 /var/log/messages 디렉토리임
