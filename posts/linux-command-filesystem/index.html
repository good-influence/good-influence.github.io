파일 시스템

파일시스템 구조

유닉스형 운영체제(리눅스), 윈도우즈에서는 계층적인 디렉토리 구조로 파일을 구성 = 트리형식 최상위에 있는 디렉토리를 루트 디렉토리 라고 한다.

윈도우는 저장장치마다 개별 파일시스템으로 관리하는 반면 유니긋형 시스템에서는 아무리 많은 저장 장치가 설치되었다 하더라도 단일 파일 시스템으로 관리한다.
유닉스형 시스템의 저장장치들은 시스템 관리자의 재량에 따라 다양한 위치에 설치(마운트)된다.

현재 작업 디렉토리

우리가 위치해 있는 지점을 현재 작업 디렉토리라 칭한다.

pwd 명령어를 통해 현재 작업 디렉토리를 알 수 있다. (Pring Working Directory)

[me@linux ~]$ pwd
/home/me

시스템에 처음 로그인하면 (또는 터미널 에뮬레이터 세션을 시작하면) 홈 디렉토리가 현재 작업 디렉토리가 된다.

디렉토리 목록 표시

현재 작업 디렉토리에 있는 파일과 하위 디렉토리를 표시할 때는 ls 명령어를 사용한다.

[me@linux ~]$ ls
Desktop Documents Music Pictures Public Templates Videos

현재 작업 디렉토리 변경

cd 경로명 으로 현재 작업중인 디렉토리를 변경할 수 있다.
경로명이란 우리가 원하는 디렉토리까지 파일시스템 트리상의 이동 경로를 나타낸 것이다.

  • 절대 경로명

    루트 디렉토리에서 원하는 디렉토리 또는 파일까지의 경로에 대하여 각 디렉토리 위치들 하나하나를 명시해준 것
    루트디렉토리는 경로명의 맨 처음을 슬래시(/)로 입력하여 표시한다.

    [me@linux ~]$ cd /usr/bin
[me@linux ~]$ pwd
/usr/bin

  • 상대 경로명

    상대경로명은 현재 디렉토리가 시작점이 된다. 상대 경로명을 표시하려면 파일시스템 트리상의 상대적인 위치를 대신 표현해주는 특수기호로 .(점)과 ..(점점)을 사용한다.
    . 기호는 현재 작업 디렉토리를 나타내고 .. 기호는 작업 디렉토리의 상위 디렉토리를 의미한다.

    현재 작업 디렉토리가 /usr/bin이라 가정하고 부모 디렉토리인 /usr로 현재 작업 디렉토리를 변경하려면 두 가지 방법을 사용할 수 있다.

    1. 절대 경로명 사용

      [me@linux ~]$ cd /usr
[me@linux ~]$ pwd
/usr

    2. 상대 경로명 사용

      [me@linux ~]$ cd ..
[me@linux ~]$ pwd
/usr

    동일하게 /usr 에서 /usr/bin 디렉토리로 이동할 때도 두 가지 방법을 사용할 수 있다.

    1. 절대 경로명 사용

      [me@linux ~]$ cd /usr/bin
[me@linux ~]$ pwd
/usr/bin

    2. 상대 경로명 사용

      [me@linux ~]$ cd ./bin  # 현재 자신 위치(.)
[me@linux ~]$ pwd
/usr/bin

      거의 모든 경우 ./ 기호를 생략할 수 있다.

      [me@linux ~]$ cd bin
[me@linux ~]$ pwd
/usr/bin

  • cd : 작업 디렉토리를 홈 디렉토리로 변경

    예를 들어 현재 작업 디렉토리가 /usr/bin이었다면 cd 명령어로 홈 디렉토리 (ex./home/me)로 변경이 가능하다.

  • cd - : 작업 디렉토리를 이전 작업 디렉토리로 변경

  • cd ~username : [username]의 홈 디렉토리로 작업 디렉토리를 변경

    예를 들어 cd ~yesol을 입력하면 사용자 yesol의 홈 디렉토리가 작업 디렉토리로 변경된다.

기타?

  • ls -a 명ㅇ령어로 .(마침표)로 시작하는 파일명을 가진 파일들도 볼 수 있다.

  • 리눅스에서는 파일명과 명령어는 대소문자를 구별한다. (File1 과 file1은 다른파일)

  • 리눅스에서는 파일 확장자 개념이 없기 때문에 원하는대로 파일명을 만들 수 있다.

  • 리눅스에서는 띄어쓰기와 구두점 기호가 포함된 긴 파일명이 허용된다. (단, 구두점 기호를 사용할 때 마침표, 대시 및 언더라인만 사용 가능) 또한 파일명에 공백을 포함하지 않는 것 이 좋다.
