Linux Command Line - How to use

Jun 5, 2020 · 3 min read

What is Command ?

  • 명령어란 /usr/bin 디렉토리에서 본 파일들처럼 __실행프로그램__을 말한다.

  • 명령어란 쉘에 내장되어 있는 명령어다. bash는 쉘-빌트인(shell builtins)이라고 하는 다수의 명령어를 내부적으로 지원한다. cd 명령어가 바로 그런 예다.

  • 명령어란 쉘 함수다.

  • 명령어란 별칭이다. 다른 명령어로부터 명령어를 새롭게 정의할 수 있다.

명령어 확인

type

쉘에 내장된 형식으로 명령어 이름을 입력하면 쉘이 실행하게 될 명령어가 어떤 타입인지를 보여준다.

[me@linux ~]$ type type
type is a shell builtin
[me@linux ~]$ type ls
ls is aliased to `ls --color=tty`
[me@linux ~]$ type cp
cp is /bin/cp

which

실행 파일의 위치를 표시해준다.

[me@linux ~]$ which ls
/bin/ls

which 명령어는 오직 실행 프로그램만을 대상으로 한다. 실행 프로그램을 대신하는 빌트인이나 별칭에는 동작하지 않는다. 쉘 빌트인(ex. cd)에 사용하면 오류메시지를 보게 된다.

명령어 도움말 보기

help

bash에는 각 쉘 빌트인마다 내장된 도움말 기능이 있다. 이 도움말을 확인하려면 다음과 같이 help를 쉘 내장 명령어 이름 앞에 입력하면 된다.

[me@linux ~]$ help cd
cd: cd [-L|-P] [dir]
Change the current directory to DIR. The variable $HOME is the default DIR. The variable CDPATH defines the search path for the directory containing DIR.

[-L|-P] 대괄호 안의 내용은 옵션이라는 것을 뜻하고 | 기호는 둘 중에 하나의 항목만을 사용한다는 것을 뜻한다.

–help

명령어 문법과 옵션에 대한 설명을 보여준다.

man

command line 용 실행 프로그램 대부분은 매뉴얼 또는 man 페이지락 불리는 공식적인 프로그램 설명서를 제공하고 있다. man program 형태로 사용한다. 일반적으로 제목, 명령어 문법 개요, 명령어 사용 목적, 그리고 명령어 옵션에 대한 설명 정도가 들어있다.

  • man 페이지 구조

    섹션 내용
    1 사용자 명령어
    2 커널 시스템 콜 API
    3 C 라이브러리 API
    4 장치 노드 및 드라이버와 같은 특수 파일
    5 파일 포맷
    6 스크린 세이버와 같은 게임이나 미디어 파일
    7 그 외 여러 종류
    8 시스템 관리용 명령어

    예제:

    [me@linux ~]$ man 5 passwd

    /etc/passwd 파일의 포맷이 무엇인지 설명하고 있는 man 페이지를 보여준다.

apropos

apropos 검색어 명령어로 man 페이지 목록을 검색할 수 있다. man 명령어를 -k 옵셕과 함께 사용하는 것과 동일한 기능이다.

whatis

특정 키워드에 부합하는 man 페이지에 대하여 그 이름과 한 줄의 간략한 정보를 보여준다.

info

man 페이지의 대안으로 GNU 프로젝트가 제공한다. info 라는 읽기 프로그램으로 볼 수 있다. 웹 페이지처럼 하이퍼링크로 되어 있다.

  • info 명령어
    명령어 실행
    ? 명령어 도움말 보기
    PAGE UP 또는 BACKSPACE 이전 페이지 보기
    PAGE DOWN 또는 SPACEBAR 다음 페이지 보기
    n 다음 - 다음 노드 보기
    p 이전 - 이전 노드 보기
    u 위로 - 현재 표시된 노드의 상위노드 (주로 메뉴) 보기
    ENTER 현재 커서 위치에 있는 하이퍼링크로 이동하기
    q 종료하기

지금까지 나온 command line 프로그램 대부분은 GNU 프로그램의 coreutils 패키지으 일부분이다 다음 명령어로 더 많은 정보를 얻을 수 있다.

[me@linux ~]$ info coreutils

README 및 기타 프로그램 문서 파일들

사용자 컴퓨터에 설치된 대부분의 소프트웨어 패키지들은 문서 파일을 가지고 있는데 /usr/share/doc 디렉토리 내에 존재한다. 대부분은 텍스트 파일 형식으로 저장되고 less명령어로 볼 수 있고 일부 파일들은 HTML 포맷으로 되어 있어 웹 브라우저를 통해서도 확인 가능하다.
.gz 확장자를 사용하는 파일들은 gzip 압축 프로그램으로 압축된 파일로 zless 라고 불리는 프로그램으로 볼 수 있다.

기타 Tip

  1. 세미콜론으로 각 명령어를 구분하여 한 줄에 하나 이상의 명령어를 입력하는 것이 가능하다. cd /usr; ls; cd-

  2. alias를 사용해서 명령배열 또는 명령을 새로운 명령어로 만들 수 있다. 이 명령어 구조는 `ailas name= ‘string’ 이다.

    1. type 명령어를 사용하여 만들 이름이 기존에 사용되고 있는지 확인한다.

      [me@linux ~]$ type test
bash: type: test: not found

      위와 같이 나오면 사용 가능한 이름이다.

    2. 별칭 만들기

      [me@linux ~]$ alias test='cd /usr; ls; cd-'

마찬가지로 별칭을 확인하기 위해 type을 사용할 수 있다. 별칭을 삭제하려면 unalias 명령어를 사용한다. 사용자 환경에 정의된 모든 별칭을 확인하려면 별도의 인자 없이 alias 명령어를 사용하면 된다.
커맨드 라인에서 별칭을 만들면 쉘 세션이 종료될 때 별칭도 사라진다.. 로그인시 환경을 설정하는 파일에 별칭 추가하는 방법 다음에 추가할게용~~
